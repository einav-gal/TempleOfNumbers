# TempleOfNumbers

## Project Overview

TempleOfNumbers is a premium web-based educational point-and-click escape room game for 6th grade students.

Technology stack:

- Phaser 3
- TypeScript
- Vite

This project is developed incrementally using small sprints.

---

# Mission

Build a polished adventure game that teaches mathematics through exploration rather than traditional questions.

The player should feel like an explorer inside an ancient magical temple.

---

# Development Rules

Always implement **one sprint only**.

When the requested sprint is complete:

STOP.

Never continue building additional systems.

Wait for the next instruction.

---

# Code Principles

Keep the project modular.

Never duplicate code.

Prefer reusable components.

Keep files small and readable.

Use TypeScript best practices.

---

# Art Principles

The project uses layered artwork.

Each room is built from:

- Background
- Objects
- Effects
- UI

Never merge assets together.

Treat every object as an independent game asset.

---

# Gameplay Principles

Never invent gameplay.

Never invent puzzles.

Never create rooms that were not requested.

Never add features that were not explicitly requested.

---

# Asset Rules

Never rename existing assets.

Use the filenames exactly as they exist.

If an asset is missing, ask before replacing it.

---

# Sprint Rules

Every sprint must produce a visible improvement.

Examples:

✔ New scene

✔ Crystal animation

✔ Inventory

✔ Popup

✔ Navigation

Not multiple systems together.

---

# Architecture

Always preserve the existing architecture.

Do not refactor unrelated systems.

Do not move files unless requested.

---

# Quality

Prioritize quality over speed.

Animations should be subtle.

Lighting should feel magical.

The atmosphere should always feel premium.

---

# Before finishing a sprint

Verify:

- Project builds
- No TypeScript errors
- No broken assets
- No console errors

Then stop.
# Golden Rule

If there is uncertainty, ask.

Never guess.

Never invent.

Never continue after completing the requested sprint.
